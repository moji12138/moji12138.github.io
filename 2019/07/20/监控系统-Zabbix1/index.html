监控系统-Zabbix(一)

发表于 2019-07-20 | 分类于 监控系统

Zabbix功能特性

Zabbix 有常见的商业监控软件所具备的功能，如主机的性能监控、网络设备性能监控、数据库性能监控、FTP等通用协议监控、多种告警方式、详细的报表图标绘制、分布式、可扩展能力、API等
数据收集
可用、性能检测
支持Agent、SNMP（包括Trapping和Polling）、IPMI、JMX、SSH、Telnet等
自定义的检测
自定义数据收集的频率
服务器端/代理端和客户端模式

灵活的触发器
可以定义非常灵活的告警阈值和多种告警相关联的条件

高度可定制的告警
发送通知，可定制包括告警级别、动作升级、收件人和媒体类型
通知可以使用全局宏变量和自定义的变量
自动处理功能包括远程命令的自动调用和执行

实时的绘图功能
监控项将数据实时绘制在图形上

Web 监控能力
Zabbix 可以模拟浏览器请求一个网站，并检查返回值和响应时间

多种可视化的展示
可以自定义监控的展示图，将多种监控数据集中展示到一张图中
网络拓扑图
自定义的 Screens和Slide shows可以将多种图形集中展示
报表功能
资源使用情况的监控展示

历史数据的存储
数据存储在数据库中
历史数据的存放周期可配置
定期删除过期的历史数据

配置简单
添加设备
应用模板

使用模板
模板可以分组
模板具有可继承性

网络发现
支持自动发现网络设备和服务器
Agent自动注册
支持自动发现实现动态监控项的批量监控，内置的自动发现包括文件系统、网络接口、SNMP OID，可定制自动发现

快速的访问接口
Web 页面基于PHP
远程访问
日志审计

API功能
应用API功能可以方便地和其他系统结合，包括手机客户端的使用

系统权限
不同的用户展示监控的资源不同
对用户的身份认证

大型环境的支持
利用Zabiix-Proxy方式即可轻松构建远程监控

Zabbix架构

Zabbix的通用架构是Client/Server架构，分布式架构为Client/Proxy/Server或Client/Node/Server，Zabbix-Server将采集到的数据持久地存储到数据库中，用前端UI友好地展示给用户。

服务器端安装(监控端)

  1. 安装Web环境

  2. 下载安装Zabbix官方源
    官网 https://www.zabbix.com/
    点击官网首页右上角的download进入下载页面
    选择下载版本-选择操作系统发行版-选择系统版本-选择准备给Zabbix所使用的数据库类型
    选择完毕后下方会自动给出安装官方源的命令

  3. 安装zabbix-server(zabbix服务)、 zabbix-web(zabbix Web前端)、zabbix-agent(zabbix客户端)

  4. 在数据库上创建zabbix数据库并进行授权

  5. 导入数据表

  6. 配置zabbix-server

  7. 启动服务

  8. 安装并配置zabbix
    浏览器访问 http://zabbix服务器地址/zabbix 开始安装

    检查环境 所有项为OK状态点击Next Step

    配置数据库信息

    配置Zabbix服务

    检查配置信息 检查之前配置是否正确，如果有误点击back返回上一步

    安装完成 如果配置有误，可在/ect/zabbix/web/zabbix.config.php中修改相关配置

    登录界面 默认管理员用户名Admin 密码zabbix

    主界面

    可修改语言

    客户端安装(被监控端)

  9. 下载安装Zabbix官方源
    和Zabbix-Server安装时使用同样方法 安装yum源

  10. 安装zabbix-agent(zabbix客户端)

  11. 配置zabbix-agent

  12. 启动服务

添加主机

添加主机组

目的:便于管理主机
配置->主机组->创建主机组->输入组名->更新 添加完成

添加被监控主机

配置->主机->创建主机

模板

什么是模板

监控项、触发器、图表、聚合图表、自动发现规则、Web监控的集合
Zabbix内置了很多适合各种需求模板
修改模板中的项目 所有使用该模板的主机的各项也会随之改变
配置->模板->创建模板->创建符合自己需求的模板

使用模板

主机名称->模板->选择模板->添加->更新
可同时选择多个模板 然后添加

稍等后可看到 可用性为绿色表示状态良好 红表示无法连接 检查agent端配置是否有误

添加监控项

主机->主机名右边的监控项->创建监控项(此修改仅对当前主机生效)
在模板中添加监控项对所有使用该模板的主机生效
查看所有监控项的详细介绍 官方文档-agent-items

自定义监控项

以剩余内存为例

在客户机添加自定义参数

使用sed、wak、grep等各种命令获得需要的数据

修改agent配置文件

重启agent服务

在Server添加自定义监控项

在server测试接收数据

同添加内置监控项一样 只是键值处要写自己自定义的参数名

图表

数据可视化 更直观的查看数据的趋势 预知未发生的风险
添加图形
配置->主机->要添加图形的主机->图形->创建图形

查看图形
监测->图形->选择主机组->选择主机->选择要查看的图形

聚合图表

同时查看多个图表
创建
监测->聚合图形->全部聚合图形->创建聚合图形

点击创建好的聚合图形名称->编辑聚合图形
点击change即可修改该区域显示的图表

选择要显示的图表

编辑各分区

查看 监测->聚合图形->全部聚合图形->可切换查看的图形

拓扑图

创建zabbix监控系统的拓扑图 还可绑定触发器在拓扑图上显示相关信息等

添加触发器

为某个监控项设置一个阈值 达到该条件便通知问题或配合Action 执行某些动作
添加
配置->主机->需要添加触发器的主机->触发器->创建触发器

查看 主面板 或 监测->问题

Zabbix报表

查看Zabbix监控系统的状态，可用性，触发器，操作日志，动作日志等统计报表

告警

告警媒介

邮件告警
利用网络发送报警信息电子邮件给相关负责人
微信告警(脚本告警)
利用企业微信API 发送通知给相关负责人
本质:执行某脚本来告警
利用此机制可完成钉钉告警或短信服务商的API来告警
SMS告警
利用GSM设备发送短信 成本较高
Jabber告警
Jabber一个开源的即时通信(IM)软件
